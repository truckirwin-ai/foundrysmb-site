# Toolkit Implementation Calendar

Ten working days to turn the free toolkit into a real business asset.

## Day 1: Pick the workflow

Output:

- three candidate workflows
- scored workflow screen
- one selected workflow

Pass test:

- the chosen workflow has a named owner, visible cost, examples, and a reviewable output

## Day 2: Map current state

Output:

- current workflow map
- inputs/tools/people list
- failure points
- review points

Pass test:

- someone who does the work agrees the map is accurate

## Day 3: Gather examples

Output:

- three good examples
- one bad or painful example
- current template/SOP if it exists

Pass test:

- the examples are specific enough for AI to learn format, rules, and edge cases

## Day 4: Build context

Output:

- business facts file
- offer/service rules
- exclusions and boundaries
- voice notes

Pass test:

- an AI output can use real company facts instead of generic advice

## Day 5: Create first prompt chain

Output:

- intake prompt
- analysis prompt
- draft/output prompt
- review checklist

Pass test:

- the prompt chain produces a useful first draft from one real example

## Day 6: Write the SOP

Output:

- workflow SOP
- review gate
- escalation rule
- change log

Pass test:

- a teammate can follow the SOP without needing the owner to explain every step

## Day 7: Test three examples

Output:

- test results
- prompt fixes
- missing context list
- pass/fail notes

Pass test:

- output improves after correction and no unsafe output gets through review

## Day 8: Measure value

Output:

- current time cost
- expected time saved
- quality/risk improvement
- monthly value estimate

Pass test:

- there is a business reason to keep improving the workflow

## Day 9: Decide implementation path

Output:

- self-install plan
- sprint target
- install backlog
- managed ops candidate list

Pass test:

- the owner can decide stop, self-run, sprint, install, or managed ops

## Day 10: Operationalize

Output:

- owner assigned
- weekly review time scheduled
- vault update rule
- measurement dashboard or spreadsheet

Pass test:

- the workflow will be used next week, not just admired as a document

## Weekly operating cadence

Every week, ask:

- Did the workflow run?
- Where did it fail?
- What context was missing?
- What prompt needs improvement?
- What output saved time?
- What still requires owner judgment?
- Is this now worth a sprint, install, or managed ops?
