Example workflow

Contractor bid desk: from lead notes to reviewed proposal draft.

Input

Lead intake

Website inquiry, photos, scope notes, customer constraints, location, due date, and past similar jobs.

AI-assisted step

Scope questions

Generate missing questions, risk flags, project assumptions, and owner-review notes before pricing.

Operator review

Estimate packet

Human confirms scope, materials, exclusions, timeline, and approval language before the proposal goes out.

Output

Follow-up path

Email draft, CRM next step, call note, and reminder sequence so the lead does not disappear.

Artifacts

The sprint package has to be usable by the operator.

01

Workflow map

Inputs, outputs, owners, handoffs, failure points, and review gates.

02

ROI readout

Hours recovered, response speed, avoidable errors, and revenue capacity.

03

Prompt chain

Reusable prompts with required inputs, expected outputs, and rejection checks.

04

SOP

Step-by-step operating procedure a teammate can follow without a consultant in the room.

05

Prototype notes

Spreadsheet, assistant, template, automation outline, or lightweight tool instructions.

06

Install decision

Stop, run the prototype, install it, or move into Managed Ops if recurring support is justified.

Pass test

A deliverable only counts if it changes the next workday.

Good sprint output lets the owner run a real example, inspect the result, correct the system, and decide whether the workflow deserves installation.

Friday acceptance check
QuestionPass condition
Can we run it?The workflow has clear inputs, steps, review gates, and outputs.
Can we judge it?The owner can compare the result against a real example.
Can we improve it?The SOP and prompt chain show where fixes belong.
Can we decide?The final packet recommends stop, self-run, install, or managed support.
