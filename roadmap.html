What qualifies

If it costs time, delays revenue, or creates errors, it belongs in the sprint.

01

Bids and proposals

Estimate prep, scope notes, past work, and follow-up need a repeatable operating path.

02

Lead follow-up

New inquiries need triage, research, message drafts, owner review, and a next action.

03

Reports and updates

Recurring summaries and client updates should not be rebuilt by hand every week.

04

SOPs and context

Expert knowledge needs capture, examples, guardrails, and retrieval structure.

Deliverables

You leave with artifacts, not advice.

01

Workflow map

Current path, failure points, inputs, outputs, owners, and review gates.

02

ROI estimate

Hours recovered, response time reduced, error risk lowered, or revenue capacity added.

03

Working prototype

Prompt chain, template, spreadsheet, SOP, assistant, or automation outline tested on examples.

04

Operating packet

SOP, prompts, source checklist, tool notes, and usage guardrails.

05

Install proposal

A clear decision to stop, run the prototype, or install the full system.

06

Handoff

The owner knows how to run the workflow after the sprint closes.
