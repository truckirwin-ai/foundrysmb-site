# AI Revenue Sprint Delivery Checklist

## Before kickoff

- Confirm payment or written payment agreement.
- Confirm primary workflow owner.
- Confirm sprint calendar and daily review window.
- Collect 3-5 real workflow examples.
- Collect current tools, templates, docs, and systems.
- Define what would make the sprint worth $1,500.
- Define what must not change for operational or legal reasons.

## Day 1: Diagnose

- Interview workflow owner.
- Capture current workflow from trigger to done.
- Identify handoffs, delays, rework, missing context, and tools used.
- Estimate time cost, missed revenue, or quality risk.
- Select one narrow sprint target.

Pass test: one-page workflow brief exists with target outcome, owner, examples, and constraints.

## Day 2: Map

- Draft current-state workflow.
- Draft improved workflow.
- Mark human decisions, AI-assisted steps, automation candidates, and review gates.
- Define required context vault structure.
- Confirm the improved workflow with client.

Pass test: client can explain the new workflow back in plain language.

## Day 3: Build

- Build prompt pack, intake format, SOP, automation outline, spreadsheet, or tool prototype.
- Connect sample data where practical.
- Create operator instructions.
- Create test cases from real examples.

Pass test: prototype produces a usable output from at least one real example.

## Day 4: Test

- Run 3-5 real examples.
- Record failure modes.
- Tighten prompts, fields, review steps, and edge cases.
- Confirm what belongs in system install versus sprint artifact.

Pass test: prototype creates useful outputs for most normal examples and has clear handling for exceptions.

## Day 5: Handoff

- Deliver final artifact package.
- Walk through owner workflow.
- Document maintenance instructions.
- Present ROI estimate.
- Present optional AI System Install proposal.

Pass test: client has the files, knows the operating path, and can decide whether to install the full system.
